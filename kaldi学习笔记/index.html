1、运行方式

执行./run.sh即可，不过在此之前，需要把kaldi/tools/env.sh添加到kaldi/tools/extras/下面，因为在yesno/s5/path.sh指定得env.sh是指定在这个位置。

2、shell脚本详解

#!/bin/bash

# 指定机器运作方式，这里是本地多任务处理工作方式，而不是gpu
# 这个文件有perl编写，可以设置核心数等参数
train_cmd="utils/run.pl"
decode_cmd="utils/run.pl"

# 下载数据集并解压
if [ ! -d waves_yesno ]; then
  wget http://www.openslr.org/resources/1/waves_yesno.tar.gz || exit 1;
  # was:
  # wget http://sourceforge.net/projects/kaldi/files/waves_yesno.tar.gz || exit 1;
  tar -xvzf waves_yesno.tar.gz || exit 1;
fi

# 解压后waves_yesno内容如下:
:<<$$
0_0_0_0_1_1_1_1.wav 等等
总共60个文件，采样率8k，wav文件中0代表读no，1代表读yes，每个文件中有8个发音
$$


# 确定训练语料和测试语料（在data目录下）
train_yesno=train_yesno
test_base_name=test_yesno

# 删除已有的数据，重新开始
rm -rf data exp mfcc

# Data preparation

local/prepare_data.sh waves_yesno
# 内部实现代码：
:<<$$

1、这个脚本先根据传入的wav文件目录，创建了waves_all.list，保存了所有waves名字
2、然后根据list，划分测试机和训练集，生成了waves.test 和 waves.train
    (注意，wav文件是不用动的，只需要把文件名写入到waves.test和waves.train中即可)
3、创建scp文件，scp文件保存了发音id和文件路径
4、创建txt文件，txt文件会把发音id转化为标签(yes no)
5、然后把上去再local文件下生成的文件，转到对应的train和test文件下，并生成utt2spk、spk2utt。
    （这两个文件都记录文件与说话人的关系）
    #!/bin/bash
    # 谁调用当前脚本，那么文件夹就在调用者的目录下为相对目录
    # 虽然当前.sh文件在local下，但是调用着run.sh在s5/目录下，
    # 所以data/local在s5下面
    mkdir -p data/local
    local=`pwd`/local
    scripts=`pwd`/scripts

    # 导入语言模型需要的执行文件
    export PATH=$PATH:`pwd`/../../../tools/irstlm/bin

    echo "Preparing train and test data"

    # 创建训练和测试文件夹
    train_base_name=train_yesno
    test_base_name=test_yesno
    # 传进来的是waves_yesno目录
    waves_dir=$1

    # -1是把waves_dir目录下的文件按行显示（去掉各种/n,/r等）
    # waves_all.list 为wav下所有wav的名字列表
    ls -1 $waves_dir > data/local/waves_all.list

    cd data/local

    # 创建test和train文件，后两个文件为输出文件，测试和训练各30个文件
    ../../local/create_yesno_waves_test_train.pl waves_all.list   waves.test waves.train

    # 创建scp文件，scp文件保存了
    # 0_0_0_0_1_1_1_1 waves_yesno/0_0_0_0_1_1_1_1.wav 
    # 发音id和文件位置
    ../../local/create_yesno_wav_scp.pl ${waves_dir} waves.test > ${test_base_name}_wav.scp
    ../../local/create_yesno_wav_scp.pl ${waves_dir} waves.trai  n > ${train_base_name}_wav.scp

    # 创建txt文件，把发音id转化为YES和NO
    # 0_0_0_0_1_1_1_1 NO NO NO NO YES YES YES YES

    ../../local/create_yesno_txt.pl waves.test > ${test_base_name}.txt
    ../../local/create_yesno_txt.pl waves.train > ${train_base_name}.txt

    cp ../../input/task.arpabo lm_tg.arpa

    cd ../..

    # This stage was copied from WSJ example
    for x in train_yesno test_yesno; do
      mkdir -p data/$x
      cp data/local/${x}_wav.scp data/$x/wav.scp
      cp data/local/$x.txt data/$x/text
     # 把text中第一列发音id作为第一列，说话人为global 
      cat data/$x/text | awk '{printf("%s global\n", $1);}' > data/$x/utt2spk
     # 输入utt2spk文件，输出每个人说了的话，这里只有一个global说的，所以只有一行
      utils/utt2spk_to_spk2utt.pl <data/$x/utt2spk >data/$x/spk2utt
    done

# 此外可能还会有以下文件：
    segments: 包括每个录音的发音分段/对齐信息（只有在一个文件包括多个发音时需要）
    reco2file_and_channel 双声道录音情况下使用到
    Spk2gender: 说话人和性别建立联系，用于声道长度归一化

# 文件目录：
    data/train_yesno/
    data/test_yesno/
    目录结构如下:
    data
    ├───train_yesno   训练文件夹
    │   ├───text      (发音id 发音文本)
    │   ├───utt2spk   (发音id 发音人)
    │   ├───spk2utt   (发音人 发音id)
    │   └───wav.scp   (发音id 发音文件)
    └───test_yesno
        ├───text
        ├───utt2spk
        ├───spk2utt
        └───wav.scp

$$


local/prepare_dict.sh
:<<$$

该文件生成的五个文件
lexicon.txt，完整的词位-发音对（所有，包括SIL）
lexicon_words.txt，单词-发音对
silence_phones.txt， SIL发音
nonsilence_phones.txt，语言学发音
optional_silence.txt ，备选SIL发音

    #!/bin/bash

    mkdir -p data/local/dict

    # 报none-silence写入到lexicon_words中
    # 这里例子只有yes和no，假设yes和no分别只有一个发音Y和N
    cp input/lexicon_nosil.txt data/local/dict/lexicon_words.txt

    # 将所有发音写入到lexicon中，包括silence的发音
    # 这个例子中的silence只有SIL
    cp input/lexicon.txt data/local/dict/lexicon.txt

    # 本例中phone只有Y、N、SIL三个，
    # grep -v反向选择，只选择非silence的phone

    cat input/phones.txt | grep -v SIL > data/local/dict/nonsilence_phones.txt

    echo "SIL" > data/local/dict/silence_phones.txt

    echo "SIL" > data/local/dict/optional_silence.txt

    echo "Dictionary preparation succeeded"

$$

# 把字典生成kaldi能够接收的数据结构FST(finit state transducer)

# 特征抽取
# 主要是提取出MFCC
:<<$$
    steps/make_mfcc.sh --nj <N> <DATA_DIR> <LOG_DIR> <MFCC_DIR> 
--nj <N>是处理器单元数
<DATA_DIR>训练语料所在目录
<LOG_DIR>这个目录下记录了make_mfcc的执行log
<MFCC_DIR>是mfcc特征输出目录

生成的文件包括：
cmvn_test_yesno.ark   raw_mfcc_test_yesno.1.ark
cmvn_test_yesno.scp   raw_mfcc_test_yesno.1.scp
cmvn_train_yesno.ark  raw_mfcc_train_yesno.1.ark
cmvn_train_yesno.scp  raw_mfcc_train_yesno.1.scp

其中scp是发音id和绝对路径  帧数*13
cmvn是归一化倒谱系数，包括能量和一阶差分和二阶差分，共计40维
13*3+1（能量）=40

查看ark的数据，可以使用如下命令：
/home/kcpu/kaldi/src/featbin/copy-feats ark:raw_mfcc_train_yesno.1.ark ark,t:-
$$
